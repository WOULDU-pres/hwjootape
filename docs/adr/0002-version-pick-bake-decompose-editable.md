# ADR-0002 — 덱 생성: "AI 통짜 디자인 N버전 → 사용자 선택 → 분해 → 편집가능 .pptx"

- Status: Accepted (2026-06-27)
- Supersedes: [ADR-0001](0001-llm-layout-over-bake-decompose.md) (LLM-레이아웃 + 큐레이션 테마)
- Driver: 사용자 (`/grill-with-docs` 세션, 2026-06-27)

## Context

ADR-0001은 "통짜 bake → OCR 분해"의 lossy 왕복을 제거하려고 **gpt-5.5 레이아웃 JSON +
큐레이션 테마 + 진짜 편집 텍스트박스**로 전환했다. 한글 선명도·멀티페이지·텍스트 오배치는
해결됐으나, 실사용에서 새 문제가 드러났다:

- **결과가 "디자인 쓰레기"** — gpt-5.5 JSON + 데이터 테마는 박스/색을 채우는 수준이라
  시각적으로 빈약하고, "내용이 어떻게 들어갈지"가 사용자에게 와닿지 않는다.
- 이미지 단계가 **장식 배경 1장**에 치중해(텍스트 없는 사이드 이미지) 슬라이드의 완성형이
  전달되지 않는다.
- 사용자 핵심 욕구: **"완성된 PPT 슬라이드"를 여러 디자인 버전으로 눈으로 보고 고르고 싶다.**

즉 ADR-0001은 "왕복 lossy"는 고쳤지만 **"AI가 만든 응집된 비주얼을 보고 승인"하는 가치**를
함께 버렸다. 사용자는 그 가치를 되찾되, ADR-0001이 지적한 실패 모드는 피하길 원한다.

핵심 신규 통찰: **"AI 디자인이 쓰레기"는 1장만 뽑아서 생긴 문제다.** god-tibo를 병렬로
여러 번 불러 **N개 버전을 만들고 사람이 좋은 걸 고르면** 품질 문제가 선택으로 해소된다.
god-tibo 백엔드는 병렬 세션 호출이 가능하다(최소 8 동시 OK, 재시도로 보강).

## Decision

ADR-0001을 역전하고, approach A를 **버전 선택 + LLM 매핑 + 배경 재생성**으로 강화해 되살린다.
`/deck` 크리티컬 패스를 다음으로 교체한다(아웃라인 마크다운 입력은 유지):

1. **프리뷰 생성** — 아웃라인 + **간략 스타일 힌트만**(레이아웃은 codex 자율) 으로
   god-tibo가 **8개 버전 × 대표 샘플 3장 = 24장**을 **병렬(동시성 3→8) + 재시도**로 굽는다.
   이 단계의 한글 텍스트는 깨질 수 있으며 **placeholder(디자인 판단용)** 로만 취급.
2. **선택** — 사용자가 1개 버전을 고른다.
3. **전체 덱 생성** — 고른 3장을 **레퍼런스 이미지로** 주입해 나머지 슬라이드를 같은 룩으로
   생성(이미 만든 3장 재사용).
4. **분해(전 슬라이드)** — Apple Vision OCR + SAM3. 텍스트 영역→아웃라인 매핑은
   **gpt-5.5 LLM 매핑**(박스 위치/개수 + 아웃라인 → 제목/불릿 할당)으로, 순수 기하 휴리스틱의
   오배치를 보완. 구운 글자는 버리고 **진짜 아웃라인 텍스트를 편집가능 박스로** 얹는다.
5. **요소 자연화** —
   - 배경: god-tibo가 **clean plate를 통짜 재생성**(원본=레퍼런스, "텍스트·전경 없음").
     `clean-bg.py` 인페인팅의 "디자인 뭉갬"을 **재생성으로 대체**.
   - 객체(아이콘·일러스트·사진): SAM3 마스크 컷아웃 + god-tibo 재생성, drift 시 컷아웃 폴백
     (`regenerate.ts` 그대로).
6. **편집(웹)** — 이미지·텍스트 **이동/크기/ z순서/삭제**(더블클릭=텍스트 내용 편집).
7. **재생성** — **요소 단위**(요구 프롬프트 + god-tibo) **및 슬라이드 전체**(재굽기 + 재분해) 둘 다.
8. **Export** — **편집가능 .pptx**(배경 plate full-bleed + 객체 개별 그림 + 편집 텍스트박스)
   **+ 슬라이드별 PNG**. `build-pptx.py` / `render-png.py` 재사용.

ADR-0001 산출물(`layout-designer.ts`, `themes.ts`, `compose.ts`, `design`/`imagery`/`recompose`
라우트)은 **크리티컬 패스에서 분리**한다(파일 보존, 되돌리기 가능).

## ADR-0001이 지적한 실패 모드에 대한 대응

| ADR-0001 지적 | ADR-0002 대응 |
|---|---|
| 텍스트 오배치(기하 휴리스틱이 CJK에서 흔들림) | **gpt-5.5 LLM 매핑**으로 박스→아웃라인 할당 |
| 분해 품질 나쁨 | SAM3 객체 분해 + LLM 텍스트 매핑 |
| `clean-bg.py` 인페인팅이 원본 뭉갬 | 배경을 **god-tibo로 통짜 재생성**(인페인팅 미사용) |
| 10장 요청에 1장만 | 이미 수정됨(`parseOutline` 전 슬라이드 반환) — 유지 |
| "AI 디자인 응집 비주얼" 가치 상실(역으로 ADR-0001의 손실) | **N버전 생성 + 사람 선택**으로 비주얼 가치 복원 |

## Consequences

- (+) 사용자가 **완성형 슬라이드 비주얼을 여러 버전으로 보고 선택** — 핵심 욕구 충족.
- (+) 최종 산출물은 여전히 **편집가능 .pptx**(텍스트 진짜 폰트로 재구성 → 한글 선명).
- (+) 배경 재생성으로 분해 후에도 자연스러운 plate 확보(인페인팅 뭉갬 없음).
- (−) **비용·지연 큼**: 24장 프리뷰 + 전체 덱 + 슬라이드별 요소 재생성 = god-tibo 호출 다수
  (병렬 8 + 재시도로 완화하나 풀 런은 수 분).
- (−) **god-tibo 한계 수용**: 마스크/strength 없음 → 객체 재생성은 흔히 컷아웃으로 폴백,
  배경 재생성은 원본 구도와 100% 일치하지 않음.
- (−) ADR-0001의 테마/레이아웃 designer 투자는 **죽은(보존된) 코드**로 남는다.
- (−) 드래그/리사이즈 에디터·버전 피커·멀티버전 생성·LLM 매핑은 **신규 구현**.

## Risks / open

- 24장 병렬 god-tibo 호출의 안정성(레이트리밋/에러) → **재시도 + 동시성 캡**으로 보강, 라이브 검증.
- gpt-5.5 LLM 텍스트 매핑이 한글 OCR 깨짐 상태(텍스트 단서 약함)에서 위치/개수만으로
  충분히 정확한가 → 검증 필요(불충분 시 기하 휴리스틱 폴백).
- 배경 통짜 재생성이 "같은 룩"을 일관되게 내는가 → 레퍼런스 주입으로 완화, 검증.
- 이미지 에셋 렌더 버그(`?project=` 누락 → `/api/projects/assets` 400/404)는 본 전환과
  독립이지만 이 워크플로우에선 이미지가 전부이므로 **선행 수정**(프로젝트 ID 스레딩 + 등록 +
  `onError` 폴백).
